# DeepCausalMMM 1.0.21 Release Notes

**Released:** 2026-05-23

This minor release adds **opt-in NOTEARS DAG structure learning** while keeping the
default **upper-triangular** acyclicity mask unchanged for existing workflows.

## Highlights

### NOTEARS causal structure learning (opt-in)

Set `config['dag_mode'] = 'notears'` to learn channel ordering from data using the
smooth acyclicity penalty **h(W) = tr(exp(W ⊙ W)) − d** (Zheng et al., 2018),
optimised under an augmented Lagrangian with periodic dual updates.

- **Huber-first warmup** (`notears_warmup_epochs`) — prediction loss only, then the
  NOTEARS penalty activates via a `notears_active` gate.
- **Per-channel parent blending** in `dag_interaction()` — each channel learns how
  much to mix its own signal with DAG-driven parents.
- **Temperature-scaled edges** (`dag_temperature`) and **column-group L1**
  (`notears_group_l1`) for sharper, focused parent sets.
- **New APIs:** `h_acyclicity()`, `notears_update_duals(factor=...)`,
  `threshold_dag(eps=...)`.
- **Trainer logging:** `[NOTEARS]` warmup and dual-update lines when `verbose=True`.

Default remains `dag_mode='triangular'` — no change for existing configs.

### Bug fixes

- **Dashboard DAG plots** (`examples/dashboard_rmse_optimized.py`):
  - Network chart and `dag_adjacency.csv` use `model.threshold_dag(eps=notears_threshold)`.
  - Heatmap uses masked, temperature-scaled adjacency aligned with the forward pass.
- **Viz defaults:** `get_viz_params()` and heatmap fallbacks use
  `correlation_threshold=0.05` (was `0.65`).
- **Trainer fallbacks** for `notears_lambda1` and `notears_dual_factor` match
  `get_default_config()`.

### Tests

- `tests/unit/test_notears.py` — forward/backward smoke tests (triangular + NOTEARS),
  acyclicity scalar, dual updates, threshold pruning, warmup gate.
- `tests/unit/test_config.py` — NOTEARS config keys and defaults.

### Documentation

- Sphinx tutorial: [`docs/source/tutorials/dag_notears.rst`](../docs/source/tutorials/dag_notears.rst)
- README NOTEARS section, quickstart cross-links, v1.0.21 API notes.

## New configuration keys

| Key | Purpose |
|-----|---------|
| `dag_mode` | `'triangular'` (default) or `'notears'` |
| `notears_warmup_epochs` | Huber-only epochs before NOTEARS penalty |
| `notears_lambda1` | L1 sparsity on adjacency |
| `notears_dual_update_every` | Outer-loop cadence (epochs) |
| `notears_dual_factor` | ρ multiplier when acyclicity stalls |
| `notears_threshold` | Pruning cutoff for `threshold_dag()` |
| `dag_temperature` | Sigmoid temperature for edge weights |
| `notears_group_l1` | Column-group L1 for focused parents |
| `visualization.dag_top_n_edges` | Global cap on DAG network edges |

## Compatibility

- **Python:** 3.9 – 3.13 (CI).
- **API:** No breaking changes. Default `dag_mode='triangular'` is unchanged.
- **Saved models:** Checkpoints from 1.0.20 load without modification.

## Upgrade

```bash
pip install --upgrade deepcausalmmm==1.0.21
```

Or install from the release branch before PyPI publish:

```bash
pip install --upgrade git+https://github.com/adityapt/deepcausalmmm.git@feature/notears-v1.0.21
```

## Enable NOTEARS

```python
from deepcausalmmm.core import get_default_config

config = get_default_config()
config['dag_mode'] = 'notears'
```

After training, inspect the learned graph with `model.threshold_dag()` or the
dashboard’s `dag_adjacency.csv`. Full guide:
[DAG and NOTEARS structure learning](https://deepcausalmmm.readthedocs.io/en/latest/tutorials/dag_notears.html).
